🗂️ Schema usado nestes exemplos — Domínio: Cinema

Este domínio é diferente do simulado. Os padrões Cypher são os mesmos — só o contexto muda.

Nós:
:Ator :Filme :Diretor :Genero :Espectador
Relacionamentos:
(:Ator)-[:ATUA_EM {papel}]->(:Filme) (:Diretor)-[:DIRIGIU]->(:Filme) (:Filme)-[:PERTENCE_A]->(:Genero) (:Espectador)-[:ASSISTIU {nota}]->(:Filme) (:Espectador)-[:AMIGO_DE {desde}]->(:Espectador)

🔗 Navegação rápida

1. MATCH + WHERE básico 2. CREATE com propriedades 3. MATCH com relacionamento 4. MERGE — encontra ou cria 5. count + ORDER BY 6. DETACH DELETE 7. shortestPath 8. WITH + avg + WHERE 9. UNWIND — criar em lote 10. Traversal multi-hop
1. MATCH + WHERE — filtrar nós por propriedade
Padrão: localizar nós de um label e filtrar com condição
Fácil
Exemplo: Todos os :Filmes lançados depois de 2010, retornando título e ano ordenados por ano.
MATCH (f:Filme)
WHERE f.ano > 2010
RETURN f.titulo AS titulo, f.ano AS ano
ORDER BY f.ano
Por quê: MATCH localiza pelo label. WHERE aplica o filtro. ORDER BY ordena — sem DESC = crescente (padrão).
Fácil
Exemplo: Todos os :Atores do país "Brasil", retornando nome e país.
MATCH (a:Ator)
WHERE a.pais = "Brasil"
RETURN a.nome AS nome, a.pais AS pais
ORDER BY a.nome
2. CREATE — criar nó com propriedades
Padrão: sempre insere, mesmo se já existir — use quando tem certeza que é novo
Fácil
Exemplo: Criar um nó :Filme com titulo: "O Poderoso Chefão", ano: 1972, nota: 9.2.
CREATE (f:Filme {
  titulo: "O Poderoso Chefão",
  ano: 1972,
  nota: 9.2
})
RETURN f
Por quê: Propriedades ficam dentro de { } após o label. CREATE sempre cria — pode gerar duplicatas se chamado duas vezes.
3. MATCH com relacionamento — navegar o grafo
Padrão: encontrar nós conectados por um tipo de relacionamento
Fácil
Exemplo: Todos os :Atores que participaram do filme "Matrix".
MATCH (a:Ator)-[:ATUA_EM]->(f:Filme {titulo: "Matrix"})
RETURN a.nome AS ator, f.titulo AS filme
Médio
Exemplo: Todos os :Espectadores que assistiram algum filme, retornando nome e total de filmes assistidos, ordenado decrescentemente.
MATCH (e:Espectador)-[:ASSISTIU]->(f:Filme)
RETURN e.nome AS espectador,
       count(f) AS total_assistidos
ORDER BY total_assistidos DESC
Por quê: count() no RETURN agrupa automaticamente pelo que está fora da função. ORDER BY DESC = do maior para o menor.
4. MERGE — encontra ou cria (idempotente)
Padrão: garante que o nó existe; cria só se não existir; sempre inclua a propriedade-chave
Fácil
Exemplo: Garantir que existe um nó :Genero com nome: "Ficção Científica".
MERGE (g:Genero {nome: "Ficção Científica"})
RETURN g
Por quê: MERGE é idempotente — se rodar 10 vezes, continua existindo só 1 nó. A propriedade dentro do { } é a chave de busca.
Médio
Exemplo: MERGE com ON CREATE e ON MATCH para tratar os dois casos separadamente.
MERGE (a:Ator {nome: "Fernanda Montenegro"})
ON CREATE SET a.pais = "Brasil", a.criado_em = timestamp()
ON MATCH  SET a.ultimo_acesso = timestamp()
RETURN a
Por quê: ON CREATE só roda quando o nó é criado; ON MATCH quando já existe. Útil para registrar timestamps sem sobrescrever dados.
5. count + ORDER BY DESC — contar e rankear
Padrão: contar quantos nós de um tipo estão conectados a cada nó de outro tipo
Médio
Exemplo: Quantos :Filmes há em cada :Genero, do mais popular para o menos.
MATCH (f:Filme)-[:PERTENCE_A]->(g:Genero)
RETURN g.nome AS genero,
       count(f) AS total_filmes
ORDER BY total_filmes DESC
Médio
Exemplo: Ranking de atores pelo número de filmes em que atuaram (top 5).
MATCH (a:Ator)-[:ATUA_EM]->(f:Filme)
RETURN a.nome AS ator,
       count(f) AS total_filmes
ORDER BY total_filmes DESC
LIMIT 5
6. DETACH DELETE — remover nó e relacionamentos
Padrão: DELETE sozinho falha se o nó tem relacionamentos; DETACH DELETE remove tudo
Fácil
Exemplo: Deletar completamente o :Diretor com id = 10 e todos os seus relacionamentos.
MATCH (d:Diretor {id: 10})
DETACH DELETE d
Por quê: Primeiro MATCH para encontrar o nó; depois DETACH DELETE remove as arestas e o nó. Sem DETACH, o Neo4j lança erro se o nó ainda tiver relacionamentos.
7. shortestPath — caminho mais curto entre dois nós
Padrão: localize os dois nós separadamente, depois aplique shortestPath com [:REL*]
Médio
Exemplo: Caminho mais curto entre o espectador "Carlos" e "Fernanda" pela rede de amizades.
MATCH (a:Espectador {nome: "Carlos"}),
      (b:Espectador {nome: "Fernanda"})
MATCH path = shortestPath((a)-[:AMIGO_DE*]-(b))
RETURN path,
       length(path) AS graus_de_separacao
Por quê: [:AMIGO_DE*] significa "qualquer número de hops". Sem direção (- e não ->) navega em ambos os sentidos.
8. WITH + avg + WHERE — agregar e filtrar (equivalente ao HAVING)
Padrão: MATCH → WITH (calcula) → WHERE (filtra sobre o resultado) → RETURN
Médio
Exemplo: Filmes com nota média dos espectadores acima de 8.0, com pelo menos 2 avaliações.
MATCH (e:Espectador)-[av:ASSISTIU]->(f:Filme)
WITH f.titulo AS filme,
     round(avg(av.nota), 2) AS media,
     count(av)              AS total_avaliacoes
WHERE total_avaliacoes >= 2
  AND media > 8.0
RETURN filme, media, total_avaliacoes
ORDER BY media DESC
Por quê: Não é possível usar WHERE sobre avg() diretamente no RETURN. O WITH cria um "checkpoint" — agrega primeiro, filtra depois. Equivale ao HAVING do SQL.
Fácil
Exemplo: Diretores cuja filmografia tem duração média acima de 120 minutos.
MATCH (d:Diretor)-[:DIRIGIU]->(f:Filme)
WITH d.nome AS diretor, avg(f.duracao) AS media_duracao
WHERE media_duracao > 120
RETURN diretor, round(media_duracao, 1) AS media_min
ORDER BY media_duracao DESC
9. UNWIND — criar múltiplos nós a partir de uma lista
Padrão: UNWIND desempacota a lista e cria/encontra um nó por elemento
Médio
Exemplo: Criar nós :Genero para "Ação", "Drama" e "Comédia" a partir de uma lista, sem duplicar.
UNWIND ["Ação", "Drama", "Comédia"] AS nome_genero
MERGE (g:Genero {nome: nome_genero})
RETURN g.nome
Por quê: UNWIND transforma a lista em 3 linhas separadas. Para cada linha, o MERGE garante que não duplica. Mais eficiente do que 3 MERGEs separados.
Médio
Exemplo: Criar nós :Ator para uma lista de nomes com atributos.
UNWIND [
  {nome: "Marlon Brando", pais: "EUA"},
  {nome: "Al Pacino",     pais: "EUA"},
  {nome: "Robert De Niro",pais: "EUA"}
] AS dados
MERGE (a:Ator {nome: dados.nome})
SET a.pais = dados.pais
RETURN a.nome
10. Traversal multi-hop — navegar por vários relacionamentos
Padrão: encadeie múltiplos relacionamentos no MATCH para percorrer o grafo em profundidade
Difícil
Exemplo: Espectadores que assistiram filmes do mesmo :Genero que "Ana" assistiu, sem incluir a própria Ana. Retorne nomes distintos.
MATCH (ana:Espectador {nome: "Ana"})
      -[:ASSISTIU]->(f1:Filme)
      -[:PERTENCE_A]->(g:Genero)
      <-[:PERTENCE_A]-(f2:Filme)
      <-[:ASSISTIU]-(outro:Espectador)
WHERE outro <> ana
RETURN DISTINCT outro.nome AS espectador_indicado,
               g.nome      AS genero_em_comum
ORDER BY espectador_indicado
Por quê: Partimos de Ana e navegamos: Filme → Gênero ← Filme (inverso) ← Espectador (inverso). O WHERE filtra Ana do resultado. DISTINCT evita repetir espectadores que compartilham vários gêneros.
Difícil
Exemplo: Atores que atuaram em filmes do mesmo diretor que dirigiu filmes com o ator "Tom Hanks", sem incluir o próprio Tom Hanks.
MATCH (tom:Ator {nome: "Tom Hanks"})
      -[:ATUA_EM]->(f1:Filme)
      <-[:DIRIGIU]-(d:Diretor)
      -[:DIRIGIU]->(f2:Filme)
      <-[:ATUA_EM]-(outro:Ator)
WHERE outro <> tom
RETURN DISTINCT outro.nome AS ator_indicado,
               d.nome      AS diretor_em_comum
ORDER BY ator_indicado
